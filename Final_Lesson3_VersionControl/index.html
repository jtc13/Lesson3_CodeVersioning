Highlight #1 Version Control

The role of version control is to have one main repository for all files that can be accessed by multiple users. The point is to have one user to write something, have another edit and write more and have someone amend that and the cycle goes on and on. It is a brilliant use of file sharing.

Highlight #2 Remote Repository

Remote repositories are on remote servers and are shared between multiple end users. Like version control, the goal is to have a team of people edit a file and save it. The difference is that when these files are saved, if the file is open on another device, that device will be updated. It's called pulling information and pushing information. The updated version is pulled from the user and pushed onto the other users that are amending the file.

Highlight #3 Git

Git is a distributed version control system that is free! The purpose of git is to have files saved locally onto user's devices, preventing unintentional override when saving onto a shared server.
